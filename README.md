# isaflow-bot

키움증권 ISA 계좌의 월배당 포트폴리오를 디스코드에서 관리하는 봇.
입력한 금액을 포트폴리오 비중에 맞춰 자동으로 리밸런싱 매수하고, 계좌 현황·미체결·분배금을 디스코드에서 조회한다.

PC에서 `run.bat`을 더블클릭하면 봇이 시작되고, 디스코드 슬래시 커맨드로 사용한다.

## 명령어 한눈에 보기

| 명령어 | 기능 |
|---|---|
| `/계좌현황` | 평가금액·매수가능금액·손익·수익률·보유 종목 |
| `/매수 [금액]` | 포트폴리오 비중대로 리밸런싱 매수 (확인 버튼) |
| `/미체결` | 미체결 매수 주문을 최신 매도1호가로 재주문 |
| `/배당 [기간]` | 분배금 내역 + 다음달 예상 분배금 |
| `/포트폴리오` | 포트폴리오 구성 비율 차트 |

## 명령어 사용 설명서

### `/계좌현황`

ISA 계좌의 전반적인 상태를 한눈에 본다.
표시 항목: 총 평가금액, 총 매입금액, 매수 가능 금액, 평가손익, 수익률, 보유 종목 목록.

> "매수 가능 금액"은 미체결 주문이 차지한 금액을 제외한 실제로 매수에 쓸 수 있는 돈이다.

### `/매수 [금액]`

입력한 금액을 포트폴리오 비중에 맞춰 자동 분배 매수한다. 2단계로 동작:

1. **리밸런싱** — 기존 보유분을 고려해 목표 비중에 가장 가까워지도록 1주씩 배분
2. **잔여금 소진** — 목표를 다 채운 뒤 남은 돈으로 가장 덜 채워진 종목부터 추가 매수

매수는 **매도1호가 지정가**로 진행되며, 확인 버튼을 거쳐야 실제 주문이 나간다.
주문 후 3초 뒤 체결 결과(전량 체결 / 일부 체결 / 미체결)를 다시 보여준다.

```
예) /매수 300000
```

### `/미체결`

조회 시점과 주문 시점 사이 호가가 바뀌어 체결되지 않은 매수 주문을 일괄 정리한다.

1. 미체결 매수 내역 조회
2. 각 종목의 최신 매도1호가 조회
3. 확인 버튼 → 기존 주문 취소 → 최신 가격으로 재주문

미체결 주문이 없으면 "✅ 미체결 주문이 없습니다"로 안내하고 종료한다.

### `/배당 [기간]`

ISA 계좌의 분배금(수익분배금입금) 입금 내역을 조회한다.
표시 항목: 종목별·월별 분배금 합계, 기간 총합, 다음달 예상 분배금(참고치).

**기간 입력 형식**

| 입력 | 의미 |
|---|---|
| `/배당` | 이번 달 |
| `/배당 2026-04` | 특정 월 |
| `/배당 2026-01~2026-04` | 기간 (시작월~종료월) |
| `/배당 전체` | 전체 기간 (2020년부터) |

**다음달 예상 분배금** — 종목별 가장 최근 분배 1회 금액을 그대로 추정치로 사용한다.
분배금은 매월 변동하므로 **참고치**일 뿐, 실제 금액과 다를 수 있다.

### `/포트폴리오`

설정된 포트폴리오 구성을 비율 차트와 함께 조회한다.
1F(안전판) / 2F(월세 수입) / 3F(인컴 엔진) 그룹별 비중과 각 종목의 분배율을 표시한다.

## 디렉토리 구조

```
isaflow-bot/
├── main.py              진입점 (봇 실행)
├── run.bat              Windows 실행 스크립트 (더블클릭)
├── pyproject.toml       uv 의존성
├── .env                 비밀키 (직접 작성, git 제외)
├── assets/              봇 아이콘
├── data/portfolio.json  포트폴리오 구성
├── logs/bot.log         실행 로그
├── core/                설정·로깅·매수계획
├── kiwoom/              키움 REST API 연동
└── bot/                 디스코드 봇 + 슬래시 커맨드
```

## 설치

요구 사항: Python 3.11+, [uv](https://docs.astral.sh/uv/), 키움 REST API 사용 신청, 디스코드 계정

```bash
cd isaflow-bot
uv sync
```

## 사전 준비

**1. 키움 REST API** — [키움 OPEN API](https://openapi.kiwoom.com)에서 신청 후 앱키·시크릿키 발급.
키움 마이페이지에서 **ISA 계좌가 기본 계좌로 설정**되어 있는지 확인한다.
(키움 API는 앱키 한 쌍이 한 계좌에 매핑되며, 요청에 계좌번호를 보내지 않는다.)

**2. 디스코드 봇** — [Developer Portal](https://discord.com/developers/applications)에서 봇 생성 →
봇 토큰 발급 → OAuth2 URL Generator에서 `bot`, `applications.commands` 스코프로 본인 서버에 초대.
봇 아이콘은 `assets/icon_512.png`를 General Information과 Bot 탭에 업로드.

**3. ID 확인** — 디스코드 개발자 모드를 켜고 본인 프로필과 사용할 채널의 ID를 복사.

**4. 환경 변수** — `.env.example`을 `.env`로 복사 후 값 입력.

```
KIWOOM_APP_KEY=...
KIWOOM_APP_SECRET=...
KIWOOM_ACCOUNT_NO=...           # 표시용
KIWOOM_IS_MOCK=false            # 모의투자는 true
DISCORD_BOT_TOKEN=...
DISCORD_OWNER_ID=...            # 본인 디스코드 ID
DISCORD_ALLOWED_CHANNEL_ID=...  # 봇을 쓸 채널 ID
```

**5. 포트폴리오** — `data/portfolio.json`에 실제 ETF 종목을 작성. 종목코드(`ticker`)는
키움 계좌의 실제 코드와 일치해야 하며, 비중(`weight`) 합계는 1.0이어야 한다.

## 실행

**Windows에서는 `run.bat`을 더블클릭**하면 가장 편하다.
직접 터미널에서 실행하려면:

```bash
uv run python main.py
```

디스코드의 지정 채널에서 `/`를 입력해 명령어를 사용한다.
봇을 종료할 땐 터미널에서 `Ctrl+C`를 누르거나 창을 닫는다.
키움 토큰은 자동 발급·갱신된다.

## 처음 사용 시

1. `.env`의 `KIWOOM_IS_MOCK=true`로 모의투자에서 먼저 테스트
2. `/포트폴리오` → `/계좌현황`으로 조회 기능 확인
3. `/매수`는 소액으로 시도, 취소 버튼 동작도 확인
4. 키움 앱에서 체결 내역이 일치하는지 확인 후 `KIWOOM_IS_MOCK=false`로 전환

> 매수는 한국 주식시장 정규장 시간(평일 09:00~15:30)에만 체결된다.

## 자주 발생하는 문제

| 증상 | 해결 |
|---|---|
| 토큰 발급 실패 | 앱키·시크릿 오타, 또는 API 신청 미승인 |
| 보유 종목이 안 나옴 | 키움 마이페이지에서 ISA를 기본 계좌로 설정 |
| 슬래시 커맨드가 안 보임 | 봇 초대 시 `applications.commands` 스코프 누락 — 재초대 |
| 봇은 온라인인데 무응답 | `DISCORD_OWNER_ID` / `DISCORD_ALLOWED_CHANNEL_ID` 확인 |
| 매수 후 미체결 | 조회~주문 사이 호가 변동 — `/미체결`로 재주문 |
| 분배금이 안 나옴 | 키움이 분배금을 "수익분배금입금" 적요로 처리하는지 확인 |

자세한 오류는 `logs/bot.log`에서 확인할 수 있다.

## 기술 스택

discord.py · httpx · matplotlib · python-dotenv · uv

키움 API: `au10001`(토큰), `kt00018`(잔고), `kt00001`(예수금), `kt00015`(거래내역),
`ka10001`(현재가), `ka10004`(호가), `kt10000`(매수주문), `kt10003`(주문취소), `ka10075`(미체결)

## 보안

`.env`에는 실제 자산을 움직이는 키가 들어있으니 외부에 공유하지 않는다.
봇은 본인 계정이 지정 채널에서 쓸 때만 동작하며, 비공개 서버 운영을 권장한다.
모든 매수 명령은 `logs/bot.log`에 기록된다.

## 면책

개인용 도구이며, 투자 결정과 결과의 책임은 사용자 본인에게 있다.
실계좌 운용 전 모의투자에서 충분히 테스트할 것.
